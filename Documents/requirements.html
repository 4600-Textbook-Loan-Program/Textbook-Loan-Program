Textbook Loan Program Requirements

Team members

Jesse Pickard, Austin Jones, Dustin Zook

Purpose

A software application that allows students to borrow textbooks, track loans, and manage returns within an educational institution.

Requirements

Name: user_authentication
Type: Functional
Summary: Secure login for users and administrators.

Name: book_search
Type: Functional
Summary: shall be able to search for textbooks using title, author, or ISBN.

Name: book_reservation
Type: Functional
Summary: Students shall be able to place holds on textbooks.

Name: book_borrowing
Type: functional
Summary: Admins should be able to check books out to students and set a return date.

Name: book_return
Type: functional
Summary: Students should be able to return textbooks, and the system must update inventory.

Name: real_time_availability
Type: functional
Summary: The system must display the real time availability of textbooks.

Name: loan_history_tracking
Type: functional
Summary: Students should be able to view their current and past borrowed textbooks.

Name: admin_dashboard
Type: Functional
Summary: Provide the administrators with a dashboard that has tools for them to manage the textbook inventory and make changes to student accounts.

Name: reporting_functionality
Type: functional
Summary: Administrators should be able to generate reports on textbook inventory, active loans, and overdue books.

Name: advanced_search_filtering
Type: functional
Summary: The system must provide advanced search filtering options.

Name: database_integrity
Type: Constraint
Summary: The database must have data integrity and prevent duplicate records.

Name: development_complexity
Type: Risk
Summary: Expanding beyond the core functionality may lead to missed deadlines.

Name: loan_tracking
Type: Functional
Summary: A tracked history for students to see what textbooks they used and when they need to return them.
